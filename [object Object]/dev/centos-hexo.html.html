CentOS 6.5下Hexo的折腾记录

By DW

发表于 2015-12-26

生命在于折腾

前言

        由于自己作死，把服务器安装了CentOS 6.5，对于我这个命令行小白来说简直是一种灾难。。。算了算了，自己作的死，忍着泪也要死完。

环境需求

  • Node.js
  • Git

安装

        先用putty连接上服务器，然后安装Node.js：

# yum -y install nodejs

        安装完Node.js继续安装Git：

# sudo yum install git

        然后可以查看版本来确定是否安装成功了：

# node -v

# git --version

        OK，下面我们安装Hexo：

# npm install -g hexo-cli

        安装Hexo完成后，我这里进入/usr文件夹下并执行如下命令：

# cd /usr
# hexo init myhexo
# cd myhexo
# npm install

其中myhexo是自己指定的文件夹，你可以任意取自己喜欢的名字。

主要目录结构看起来应该是这样的：

|--- _config.yml
|--- node_modules
        |-- xx
        ·
        ·
        ·
        |-- xx
|--- package.json
|--- scaffolds
        |-- draft.md
        |-- page.md
        |-- post.md
|--- source
|--- themes

        Linux下用Vi操作文件非常不方便（至少对我这种命令行小白来说很不方便。。。），这时候我们可以安装Ftp工具，这里我安装许多人比较推崇的vsftpd。

# yum -y install vsftpd

检查一下nologin文件的位置，看看是在/usr/sbin目录下还是在/sbin目录下，下面代命令的目录根据这里来。接着我们创建名为myftp的账户。并且指定/usr为用户myftp的根目录，（此目录不要学我，我偷懒想用ftp操作底下的所有文件）。

# useradd -d /usr -s /sbin

修改我们刚刚创建的myftp账户的密码：

# passwd myftp

根据提示输入两遍密码。

接着对vsftpd进行一些配置，用vi来编辑配置文件：

# vi /etc/vsftpd/vsftpd.conf

找到并改成这样：

anonymous_enable=NO
local_enable=YES
write_enable=YES

vi中x是删除当前字符，i进入编辑模式，可以插入文本，ESC退出编辑模式进入命令模式，在命令模式下ZZ保存修改并退出。ps：命令我也是刚查的。。。

接着我们配置它开机自启动：

# chkconfig vsftpd on

启动服务：

# service vsftpd start

成功启动了，现在我们可以使用FTP工具来上传和下载服务器的文件了，我用的是FileZilla。
